# Token Analytics Dashboard 📊

A client-side React + TypeScript dashboard for exploring the JSON exports generated by the `token-performance-tuning` workflow.

Instead of reading raw JSON by hand, this app turns the exported data into charts, rankings, and drill-down views so you can quickly answer:

- Which projects are consuming the most tokens?
- Where are the biggest context spikes happening?
- Which sessions are the most expensive?
- What insights and optimization suggestions did the analysis already detect?

## What This Project Does ✨

The dashboard reads local JSON files copied into the project and visualizes:

- global token totals
- daily usage trends
- single-turn context spikes
- top projects, skills, automations, and agent sources
- high-cost sessions
- archived monthly history
- machine-generated insights and suggestions

It is currently an MVP and runs entirely in the browser. There is no backend.

## Data Files Explained 🗂️

The app expects the analysis output to be copied into `src/assets/data/`.

### Required files

- `src/assets/data/token-dashboard-data.json`
  Main summary export. This is the primary dataset used by the dashboard.

- `src/assets/data/token-history-index.json`
  Archive manifest. It lists which monthly archive files exist and how many sessions they contain.

- `src/assets/data/token-history/<month>.json`
  Monthly archive files such as `2025-12.json`. These are loaded only when the user opens that month in the Archives view.

### What each file contains

`token-dashboard-data.json` includes:

- `totals`: overall token totals and max spikes
- `daily`: per-day aggregate metrics
- `sessions`: session-level detail
- `projects`, `skills`, `automations`, `agents`: grouped rollups
- `insights`: global findings
- `suggestions`: optimization recommendations
- `trend_insights`: cross-day change signals

`token-history-index.json` includes:

- available archive months
- first and last timestamps per archive
- retention metadata

Each monthly archive file includes:

- archived session rows for that month
- lightweight metadata for the archive itself

## How To Use It 🚀

### 1. Install dependencies

```bash
npm install
```

### 2. Make sure the data files exist

Copy the latest exported JSON files into:

```text
src/assets/data/
src/assets/data/token-history/
```

### 3. Start the dashboard

```bash
npm run dev
```

Then open the local Vite URL shown in the terminal.

## Architecture Overview 🧱

The app is organized in a simple client-side structure:

```text
src/
  assets/data/          Static JSON copied from the analysis workflow
  components/           Reusable UI building blocks
    charts/             Shared chart components
    filters/            Reusable filter controls
    layout/             App shell and navigation
    states/             Loading, empty, and error states
  features/             Screen-level features
    overview/
    sessions/
    dimensions/
    archives/
    insights/
  lib/
    analytics/          Typed client-side analytics events
    data/               Loaders, schemas, normalization, selectors
    formatters.ts       Shared formatting helpers
```

### Data flow

1. JSON files are imported from `src/assets/data/`
2. Zod schemas validate the raw data
3. Normalizers convert the raw export into chart-safe structures
4. Selectors build KPI and ranking datasets
5. Feature pages render charts and detail panels

This separation keeps the UI resilient when the export shape is slightly irregular.

## Important Data Handling Notes ⚠️

- The export can contain partial daily rows for older archived periods.
  The app normalizes missing numeric values to safe defaults so charts do not crash.

- The export may contain the same `project_name` under different `project_path` values.
  The dashboard consolidates those rows when showing project-level rankings.

- Archive files are not loaded at startup.
  They are fetched only when the user explicitly selects a month.

- If a JSON file has the wrong shape, the app shows an in-app error state instead of a blank screen.

## Analytics Instrumentation 📈

It emits browser `CustomEvent`s named:


Tracked interactions include:

This is intentionally minimal and client-only for now.

## Troubleshooting 🔎

### The page is blank

Most likely a JSON parsing issue. Check that:

- `token-dashboard-data.json` exists
- `token-history-index.json` exists
- archive file names match the month values in the index
- the copied files are valid JSON

The app should now show an error card if parsing fails, which helps identify the cause.

### Archive data does not appear

Check that:

- the selected month exists in `token-history-index.json`
- the matching file exists in `src/assets/data/token-history/`
- the file name exactly matches the month, for example `2025-12.json`
